# 📖 中州韵输入法（Rime）输入法指南

这是一篇基于自己使用Rime的配置指南，旨在解决现在网络上部分教程已经过时的问题。

更新日期：`2026-03-24`

## 📋 概述

中州韵输入法（Rime）是一个跨平台的开源输入法算法框架。

因为商业原因，市场上主流的输入法越来越臃肿，普遍开始夹杂各种令人烦不胜烦的广告和植入功能。

并且在编程等情景下，普通的输入法的使用体验非常不舒适。

这时候，想必你也希望可以拥有一个开源、可自定义的输入法，但是又苦恼于如何无痛过度到新的输入法。

本指南旨在帮助您入门安装和配置这个优秀的开源输入法，并减少您在过度时的困惑。

### ⚠️ 缺点

很多配置都采用yaml或txt的形式，没有UI菜单，对于新手来说不太友好。

## 💾 中州韵输入法（Rime）的安装

官方网站：https://rime.im/

根据你使用的操作系统，下载对应的安装包（安装简单）。

如果有特殊需要，也可以去Github仓库下载对应版本。

## ❄️ 雾凇拼音（Rime-ice）的安装

雾凇拼音提供了一套开箱即用的完整配置，包含输入方案（全拼、常见双拼）、长期维护的开源词库及各项扩展功能。

github地址：https://github.com/iDvel/rime-ice

### 📥 安装方式

1. 在输入法图标上右键，在弹出的菜单中点击`输入法设定（S）`。
2. 点击弹窗左下角的`获取更多输入方案...`。
3. 在弹出的控制台中输入：`iDvel/rime-ice`
4. 控制台会自动去检索这个 pacakge name 然后自动安装。
5. 安装完成后，在方案列表中近勾选`雾凇拼音`就行了。
6. 点击`中`，完成配置。
7. 如果不需要修改皮肤，再次点击`中`完成配置，弹窗会自动关闭。

## 🔧 配置文件补丁

这部分的详细文档可以参考：https://dvel.me/posts/rime-ice/#配置的引用

1. 通过`补丁`的方式修改配置文件，可以在更新方案后，避免配置被覆盖。
2. 所有配置字段都可以通过`补丁`进行添加、修改、覆盖。
3. 但是请注意对列表或字典的覆盖是否符合预期（确定您是想为列表/字典添加成员，还是完全覆盖该容器）。
4. 在输入法图标上右键，在弹出的菜单中点击`用户文件夹（C）`，即可以打开配置文件所在的文件夹。

### 📝 配置文件示例

在项目中，有我正在使用的配置：`rime_ice.custom.yaml`。
您可以直接复制到`用户文件夹`中使用，也可以对其进行修改。

例如：找到想要激活的模糊音，为其取消注释（快捷键：`Ctrl+/`，或删除前置的`#+空格`-注意不要破坏缩进）。

### 🎵 模糊音设置

下面以追加模糊音规则为例：

1. 在输入法图标上右键，在弹出的菜单中点击`用户文件夹（C）`
2. 新建一个文件：`rime_ice.custom.yaml`（作为`rime_ice.schema.yaml`的补丁），用`VSCode`（或其他文本编辑器）打开。
3. 粘贴以下yaml配置：

```yaml
# 以 patch: 开头，后面的内容都需要缩进
patch:
  ##### 结尾的 /+ 表示在原基础上追加
  # 保留已有的配置，追加模糊音规则
  speller/algebra/+:
    ### 模糊音
    # 声母
    - derive/^([zcs])h/$1/ # zh* ch* sh* 派生出 z* c* s*
    - derive/^([zcs])([^h])/$1h$2/ # z* c* s*    派生出 zh* ch* sh*
    # 韵母
    - derive/ang$/an/
    - derive/an$/ang/
    - derive/eng$/en/
    - derive/en$/eng/
    - derive/in$/ing/
    - derive/ing$/in/
    - derive/uan$/uang/
    - derive/uang$/uan/
```

4. 保存这个配置文件，规则可能还没有生效。
5. 在输入法图标上右键，在弹出的菜单中点击`重新部署（R）`，现在规则应该已经生效。

### 🔢 候选词数量设置

与模糊音同理，如果要修改候选词数量：

1. 在`rime_ice.custom.yaml`的`patch:`的下级，增加字段：`menu/page_size: 10`，修改后的yaml配置如下：

```yaml
# 以 patch: 开头，后面的内容都需要缩进
patch:
  menu/page_size: 10 # 修改候选词数量为10个
  speller/algebra/+:
    # 省略这部分内容
```

2. 在输入法图标上右键，在弹出的菜单中点击`重新部署（R）`，现在规则应该已经生效。

## 📚 导入其他输入法的词库

相信您也是从其他主流输入法转移到Rime，因此将之前的词频词库转移到Rime就非常有必要。

如果没有导入词频词库，您会发现输入法“不懂自己”，打字时经常会需要找自己想要的词。

### 📤 导出词库

下面以搜狗输入法为例：

1. 右键输入法的图标或状态栏，找到设置（不同版本的输入法，这个词可能会不同，比如我这里显示的是`全局设置`）。
2. 在设置里找到`词库`选项。
3. 在词库中，相信您已经看到词库有`导出/备份`按钮了，点击它，可以导出为搜狗词库的bin文件。

您还需要下载一个开源软件，用于将各个输入法自己的词库格式，转换为Rime使用的txt格式。

### 🔄 词库转换

这里推荐您使用`深蓝词库转换（imewlconverter）`。
GitHub仓库：https://github.com/studyzy/imewlconverter
Release版本：https://github.com/studyzy/imewlconverter/releases/

下载您的操作系统对应的版本，比如64位的Windows操作系统就下载`imewlconverter_win-x64.zip`。

下载后是一个zip文件，其中包含了这个软件的exe文件，解压它到您的程序文件夹中，例如：`C:\Program Files\imewlconverter`。

现在我们可以开始转换词库的格式了：

1. 双击运行`深蓝词库转换.exe`。
2. 在第一个输入框中，点击右侧的`...`按钮，选择您要转换的词库原文件。
3. 如果您发现没有显示您想要转换的文件，请在弹出的选择器中，在右下角设置一下筛选器，例如：`搜狗备份词库（*.bin）`。
4. 下方的格式转换中，左侧的原始格式应该会被正确地自动识别。
5. 需要您手动选择要输出的格式，让我们找到：`Rime中州韵`，并选择它。
6. 在弹出的Rime输入法编码设置中，选择正确的编码类型和平台，例如：`拼音`和`Windows`，点击确定。
7. 等待进度条读完，点击转换按钮，将输出的文件保存。

然后我们开始导入词库：

1.  在输入法图标上右键，在弹出的菜单中点击`用户词典管理（D）`。
2.  选择`rime_ice`，在右侧点击导入文本码表。
3.  选择之前导出的词库，点击`打开`。
4.  恭喜，您已经完成了词库的导入。

## 🎯 模糊音显示正确注音

> 💡 本功能通过自定义 Lua 过滤器实现，感谢 GitHub Copilot 的智能辅助。

当您启用模糊音后（例如 zh/z、ch/c、sh/s 不分），输入法虽然能够识别出您想要的词，但您可能会不记得这个词的正确读音。这时候，`say_it_right_filter` 可以在候选词旁边自动标注正确的拼音。

### 📋 功能说明

- **自动识别模糊音**：当您使用模糊音输入时，自动检测并标注正确读音
- **智能匹配**：支持完整拼音、模糊音和首字母简写三种匹配模式
- **可自定义样式**：支持配置标注格式、连接符等显示样式
- **友好兼容**：与 `corrector` 等其他过滤器兼容，不会产生冲突

### 📝 使用示例

例如，当您配置了 `chi → c` 的模糊音规则后：

- 输入 `ci fan`（使用了模糊音）
- 候选词显示：`吃饭 〔吃=chi 饭=fan〕`
- 这样您就知道"吃"的正确读音是 `chi` 而不是 `ci`

### 🔧 配置方法

#### 1. 放置过滤器文件

将 `say_it_right_filter.lua` 文件放入用户文件夹的 `lua` 子目录中：

```
<用户文件夹>/
└── lua/
    └── say_it_right_filter.lua
```

#### 2. 在输入方案中启用

在 `rime_ice.custom.yaml` 中添加以下配置：

```yaml
patch:
  # 启用 spelling_hints（必需，用于提供真实拼音）
  translator/spelling_hints: 10

  # 在 filters 中添加 say_it_right_filter
  # 注意：必须放在 spelling_hints 之后，在 corrector 之前
  engine/filters/+:
    - lua_filter@*say_it_right_filter

  # 可选：自定义显示样式
  say_it_right_filter:
    style: "〔{pinyin}〕" # 注释外层样式
    single_char_format: "{pinyin}" # 单字格式
    multi_char_format: "{char}={pinyin}" # 多字格式
    separator: " " # 多个注释之间的连接符
    keep_original_comment: false # 是否保留原始拼音注释
```

#### 3. 重新部署

在输入法图标上右键，点击`重新部署（R）`，使配置生效。

### ⚙️ 配置选项详解

| 选项                    | 默认值             | 说明                                        |
| ----------------------- | ------------------ | ------------------------------------------- |
| `style`                 | `{pinyin}`         | 注释的外层样式，`{pinyin}` 为占位符         |
| `single_char_format`    | `{pinyin}`         | 单字的显示格式，可用 `{char}` 和 `{pinyin}` |
| `multi_char_format`     | `{char}({pinyin})` | 多字词的显示格式                            |
| `separator`             | ` `（空格）        | 多个模糊音标注之间的连接符                  |
| `keep_original_comment` | `false`            | 是否保留原始的拼音注释（兼容 corrector）    |

### 📌 注意事项

1. **依赖关系**：必须先启用 `translator/spelling_hints` 才能正常工作
2. **放置顺序**：在 filters 列表中必须放在 `spelling_hints` 之后，在 `corrector` 等修改 comment 的 filter 之前
3. **模糊音配置**：过滤器会自动读取您在 `speller/algebra` 中配置的模糊音规则（`derive` 类型）
4. **性能影响**：该过滤器对性能影响极小，可以放心使用
